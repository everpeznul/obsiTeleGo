# obsiTeleGo

Telegram-бот на Go с поддержкой очередей сообщений для пуша сообщения из Telegram (можно из бота, можно из беседы) по топикам в хранилище Obsidian.

## Стек технологий

* **Ядpо:** Go 1.22+
* **БД (опционально):** SQLite, Redis
* **Брокер сообщений:** RabbitMQ
* **Инфраструктура:** Docker, Docker Compose
* **Управление задачами:** Task (go-task)
* **Библиотека Telegram:** `github.com/go-telegram/bot`

## Требования

Для локального запуска проекта необходимы установленные в системе:
* [Docker](https://docs.docker.com/engine/install/)
* [Docker Compose](https://docs.docker.com/compose/install/) (версия v2+)
* [Task](https://taskfile.dev/installation/)

## Установка и настройка

Скопируй пример конфигурационного файла и заполни его актуальными данными:

```bash
cp config/.env.example config/.env

```

В файле `config/.env` должны присутствовать настройки для подключения к Telegram API, RabbitMQ и выбранной базе данных.

## Запуск проекта

Управление контейнерами и сборкой осуществляется через утилиту `task`. В зависимости от выбранной команды, Docker Compose поднимет нужную инфраструктуру, а компилятор Go соберет бинарный файл только с нужным драйвером БД.

Запуск бота с использованием **Redis**:

```bash
task redis

```

Запуск бота с использованием **SQLite**:

```bash
task sqlite

```

Остановка проекта (без удаления данных):

```bash
task down

```

Полная очистка проекта (удаление контейнеров, сетей, образов и томов с данными):

```bash
task clean

```

## Структура проекта

* `cmd/app/` — Инициализация приложения и внедрение зависимостей. Здесь происходит выбор репозитория через директивы `go:build`.
* `cmd/main.go` — Точка входа в приложение.
* `internal/botHandler/` — Логика взаимодействия с Telegram API, роутинг команд и обработка тредов.
* `internal/rabbitmq/` — Логика продюсеров и консьюмеров для асинхронной обработки задач.
* `internal/repository/` — Интерфейсы хранилища и их реализации (Redis, SQLite).
* `internal/obsidian/` — Интеграция с Obsidian (форматирование и клиент).
* `internal/domain/` — Доменные модели и бизнес-логика.
* `Taskfile.yml` — Описание команд для локальной разработки и сборки.
* `docker-compose.yml` — Конфигурация сервисов с использованием профилей для опционального запуска баз данных.
